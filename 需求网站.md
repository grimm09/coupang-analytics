# Coupang.JY 经营分析网站完整版需求说明

## 0. 文档目的

本文档用于定义 Coupang.JY 经营分析网站的完整产品需求、数据需求、页面模块、图表规则、导出规则与后续多月份扩展规范。

当前网站已完成 2026 年 1 月、2 月、3 月的数据接入与页面展示，后续需要基于同一套数据层和页面结构继续接入其它月份数据，例如 4 月、5 月。

本文档是交接给后续开发/制作模型的基准需求文档，不只是早期规划稿。后续制作其它月份时，必须以本文档中的数据来源、字段口径、显示规则和异常处理规则为准。

## 1. 项目定位

Coupang.JY 是一个用于分析 Coupang 店铺经营表现的个人经营数据看板。

网站核心目标：

- 统一读取不同月份、不同店铺、不同来源的数据。
- 用 Next.js + ECharts 做网页端可视化。
- 用 Matplotlib 复用同一套数据层生成月度展示图。
- 支持经营总览、产品分析、品类分析、广告分析、月度对比和报告中心。
- 为后续新增店铺、新增月份、新增产品数据预留扩展能力。

## 4. 全局显示规则

### 4.1 品牌与导航

左上角品牌名固定：

```text
coupang.JY
```

左上角品牌视觉要求：

- `coupang` 必须使用 `E:\workspace\coupang\网站左上角更改.svg` 对应的彩色图片。
- 网站内资源路径为 `public/coupang-header-logo.svg`。
- 图片只显示彩色 `coupang`，后面用文字补 `.JY`，整体视觉名称为 `coupang.JY`。
- 不要再使用旧的紫色柱状图标替代 coupang 品牌图片。

左侧导航：

- 经营总览
- 产品分析
- 品类分析
- 广告分析
- 月度对比
- 报告中心

### 4.2 金额显示规则

所有人民币金额必须带 `¥`。

正确示例：

```text
¥18.4万
¥183,570
¥27,807
¥84.16
```

不要写：

```text
人民币销售额
人民币客单价
人民币毛利润
```

字段名称只写：

- 销售额
- 客单价
- 毛利润
- 变化额

广告后台原始韩元数据可以使用 `₩`，但如果转成人民币，则必须使用 `¥`。

### 4.3 数字格式

| 数据类型 | 显示规则 |
|---|---|
| 金额 | 带货币符号，必要时显示万 |
| 订单量 | 整数，千分位 |
| 商品数 / 销量 | 整数，千分位 |
| 毛利率 | 百分比，保留 2 位 |
| 退损率 | 百分比，保留 2 位 |
| 成本占比 | 百分比，保留 2 位 |
| 环比变化 | 正数加 `+`，负数加 `-` |
| pp 变化 | 使用 `+1.90pp` 格式 |

### 4.4 产品 ID 规则

产品 ID 必须是正常 7 位数字。

禁止出现：

```text
:880002884
880002884
```

特殊已知修正规则：

```text
SL-电加热男女保暖手套 -> 1322886
```

所有产品表格展示前必须校验：

```ts
/^\d{7}$/
```

### 4.5 字体与排版

- 字体不能过小。
- 当前产品分析调整后的字体大小作为基准。
- 最终确认的经营总览 / 产品分析字号基准：
  - KPI 标签：12px。
  - KPI 主数值：`clamp(17px, 1.65vw, 24px)`。
  - KPI 辅助说明：11px。
  - 模块标题：16px。
  - 模块说明：12px。
  - 数据源标签：11px。
  - 经营总览品类洞察标签 / 数值：11px / 16px。
  - 经营总览指标栏标签 / 数值：10px / 15px。
  - 经营总览表格：12px。
  - 产品分析商品排行表格：13px。
- 数据卡片尽量横向展示，不做密集竖排。
- 表格中金额、订单、客单价、毛利率、退损率等数值应居中或右对齐，保证可读。
- 产品名称过长时使用省略号。
- 页面必须适配浏览器屏幕宽度。

### 4.6 视觉风格

整体参考 Pinterest 风格的现代数据仪表盘：

- 浅紫背景
- 白色圆角卡片
- 紫色渐变主色
- 青色 / 绿色 / 橙色作为辅助色
- 红色表示风险

## 6. 页面模块需求

## 6.1 经营总览

### 6.1.1 页面目的

经营总览用于快速判断当前月份的整体经营健康度。

当前默认展示：

```text
2026年3月经营总览
```

月份下拉当前支持：

```text
2026年1月
2026年2月
2026年3月
```

### 6.1.2 核心 KPI

必须展示：

- 订单量
- 销售额
- 客单价
- 退损率
- 毛利率

后续可扩展：

- 商品数 / 商品销量
- 毛利润
- 采购成本占比
- 物流占比
- 广告占比

### 6.1.3 图表模块

经营总览至少包含：

1. 个人与平台成本结构对比
2. 个人品类和平台品类对比饼图
3. 成本与毛利指标横排
4. 广告表现摘要
5. 品类对比明细表

#### 6.1.3.1 经营总览广告表现摘要异常规则

- 每个月生成经营总览前，必须检查该月份是否存在广告后台截图文件夹、广告后台导出表或已识别的 `adAnalysis` 数据。
- 如果该月份没有广告数据，经营总览不能静默留空，也不能沿用其它月份广告数据。
- 广告表现摘要必须明确显示异常状态，例如：

```text
3月广告后台表现：广告后台待接入
```

- 缺失广告数据时，广告表现摘要字段统一显示为 `0` 或 `待接入`，并在来源说明中写明：

```text
3月广告后台数据待接入
```

- 该异常必须同步记录到报告中心或月度报告缺失字段中，避免误判为广告表现为 0。

### 6.1.4 品类饼图要求

- 个人品类与平台品类并排。
- 饼图中心文字必须居中。
- 图例和品类文字不能重叠。
- 标签太多时使用图例或列表，不强行全部显示在饼图外圈。

### 6.1.5 自动诊断

经营总览应能给出：

- 毛利率是否偏低
- 退损率是否偏高
- 物流成本是否异常
- 采购成本占比是否异常
- 广告占比是否需要复盘

## 6.2 产品分析

### 6.2.1 页面目的

产品分析用于判断单个产品的销售能力、利润质量、退损风险、店铺分布和异常情况。

### 6.2.2 顶部 KPI

必须展示：

- 订单量
- 销售额
- 商品数
- 商品排行（销售额）

顺序必须是：

```text
订单量 -> 销售额 -> 商品数 -> 商品排行（销售额）
```

不要写：

```text
人民币销售额
```

只写：

```text
销售额
```

### 6.2.3 商品排行表格

商品排行必须包含：

- 产品 ID
- 产品名称
- 店铺
- 销售额
- 订单
- 客单价
- 毛利率
- 退损

字段命名要求：

```text
产品名称
```

不要写：

```text
商品名称
```

产品名称过长时省略号。

### 6.2.4 店铺标记

商品排行模块需要标记产品来源店铺：

- 本土88
- 本土70
- 本土88/70
- 后续新增店铺组合

需要区分：

- 单店铺出单
- 多店铺出单

示例：

```text
本土88 · 单
本土70 · 单
本土88/70 · 多
```

说明：当前页面为了节省表格宽度，使用 `单` / `多` 短标签；后续如果空间允许，也可以展示为 `单店` / `多店`，但不能丢失单店铺或多店铺出单含义。

### 6.2.5 风险产品

产品分析需要识别：

- 高退损产品
- 低毛利产品
- 亏损产品
- 销售额高但毛利低的产品
- 多店铺出单但利润异常的产品

### 6.2.6 产品矩阵

建议保留产品矩阵：

- 横轴：销售额
- 纵轴：毛利率 / 退损率
- 气泡大小：订单量
- 颜色：风险等级

- 个人品类出单订单量

模块名称：

```text
个人&平台出单品类对比
```

```text
收入 & 利润排行
```

### 6.3.4 品类饼图

必须解决：

- 标签不重叠
- 品类文字可读
- 小屏幕下不挤压

### 6.3.5 品类详情

点击品类后展示：

- 品类名称
- 个人订单
- 平台订单
- 销售额
- 毛利润
- 毛利率
- 退损率
- 采购占比
- 物流占比
- 广告占比

### 6.3.6 品类诊断

自动判断：

- 个人强于平台的品类

- 高退损品类
- 低毛利品类

## 6.4 广告分析

### 6.4.1 页面目的

广告分析用于判断广告投放表现、转化质量、ROAS、花费效率和异常周期。

### 6.4.2 数据来源

广告数据来自广告后台截图或广告后台导出数据。

页面必须保留来源说明：

```text
数据来自广告后台截图
```

#### 6.4.2.1 广告后台图片读取规则

当月份文件夹内存在广告后台截图文件夹时，广告分析模块必须优先读取截图数据，不得直接判定为“广告后台待接入”。

广告数据可能不在标准化 Excel 中，而是在原始月份文件夹的图片里。开发每个月广告模块或经营总览广告摘要前，必须检查并读取以下类型的图片文件：

```text
E:\workspace\coupang\{月份}\广告郑州吉喵云-Coupang-{店铺}\*.png
```

常见图片文件名包括：

- `店铺整体.png`
- `店铺整体广告.png`
- `店铺整体广告活动.png`
- `各广告活动.png`
- `各广告活动效果.png`
- `每周.png`
- `每周广告业绩.png`

这些图片里包含广告后台关键字段，不能因为标准化目录中没有广告 Excel 就判定广告数据缺失。

2 月已确认广告图片路径：

```text
E:\workspace\coupang\2月\广告郑州吉喵云-Coupang-本土88
```

3 月已确认广告图片路径：

```text
E:\workspace\coupang\3月\广告郑州吉喵云-Coupang-本土88
E:\workspace\coupang\3月\广告郑州吉喵云-Coupang-本土70
```

该文件夹以图片形式保存广告后台数据，当前包含三类截图：

- `广告效果/所有广告效果`：用于读取当月广告总览 KPI、总销售额、广告转化销售额、广告费、ROAS。
- `广告活动/按周表格`：用于读取每周广告表现明细，并校验月度合计。
- `每周广告业绩`：用于展示或复刻每周总销售额、广告转化销售额、广告费柱状图。

图片数据读取字段映射：

| 网站字段 | 图片字段 | 单位 | 说明 |
|---|---|---:|---|
| 曝光量 | 曝光量 | 次 | 广告后台原始曝光 |
| 点击量 | 点击量 | 次 | 广告后台原始点击 |
| 点击率 | 点击率 | % | CTR |
| CPC | 按点击次数收费 | ₩ | 后台显示为韩元 |
| 广告转化订单数 | 广告转化订单数 | 次 | 广告产生订单 |
| 转化销量 | 转化销量 | 次 | 广告转化商品销量 |
| 转化率 | 转化率 | % | CVR |
| 广告费 | 广告费 | ₩ | 广告消耗 |
| 广告转化销售额 | 广告转化销售额 | ₩ | 广告归因销售额 |

## 6.5 月度对比

### 6.5.1 页面目的

月度对比用于对比两个自然月之间的经营变化。

当前默认：

```text
2026年3月 vs 2026年4月
```

当前已支持并后续应继续支持任意相邻月份：

```text
2025年12月 vs 2026年1月
2026年1月 vs 2026年2月
2026年2月 vs 2026年3月
2026年3月 vs 2026年4月
```

### 6.5.1.1 月度对比标题显示规则

月度对比页面的可见标题必须使用中文年月，不允许把内部数据 key 直接展示给用户。

正确示例：

```text
月度对比：2026年2月 vs 2026年3月
月度对比：2026年3月 vs 2026年4月
```

不要显示：

```text
月度对比：2026-02 vs 2026-03
```

说明：数据层里 `monthA` / `monthB` 可能存在两种格式：

- `2026-02`
- `2026年2月`

前端展示时必须统一格式化为 `2026年2月` 这种中文年月。顶部导航标题、月度对比内容区标题、报告中心月度对比标题都必须保持一致。

### 6.5.2 对比口径

默认：

```text
个人产品
```

可切换：

- 个人产品
- 平台整体

### 6.5.3 顶部 KPI

必须展示：

- 收入合计
- 有效订单量
- 商品销量
- 毛利润
- 毛利率
- 退损率

其中：

- 收入合计、毛利润必须带 `¥`
- 毛利率、退损率显示 pp 变化
- 收入合计、有效订单量、商品销量、毛利润使用个人对比，不使用整体平台对比

### 6.5.3.1 顶部 KPI 金额排版规则

月度对比顶部 KPI 中，收入合计、毛利润等金额可能较长，例如：

```text
¥227,890.72
```

这些金额不能与月份标签、前一个月份数字或人民币符号发生重叠。

排版要求：

- 两个月的数值优先上下排列，避免在窄卡片内左右挤压。
- 月份标签与金额分开显示，例如左侧显示 `3月`，右侧显示 `¥227,890.72`。
- 金额必须 `white-space: nowrap`，但卡片要给足宽度或允许横向滚动。
- 若一行无法容纳，不允许压缩到互相重叠；应改为上下排列或适当缩小字号。
- 金额字段仍必须保留 `¥`，不能为了避免重叠删掉人民币符号。

## 7. 多月份数据接入规范

### 7.1 接入顺序

后续接入其它月份时，建议按以下顺序：

1. 建立该月份文件名映射。
2. 读取 Excel 与截图数据；必须显式检查广告后台截图/广告导出是否存在。
3. 转换为标准数据结构。
4. 校验产品 ID、金额、订单、比例字段。
5. 生成该月份 `overview`；如果广告数据缺失，`overview.advertising` 必须写入“广告后台待接入/数据缺失”状态，经营总览必须展示该异常。
6. 生成该月份 `productAnalysis`。
7. 生成该月份 `categoryAnalysis`。
8. 生成该月份 `adAnalysis`；如果没有广告数据，也必须生成缺失状态而不是沿用旧月份。
9. 放入 `monthlyData`。
10. 页面切换月份验证。
11. 与相邻月份生成月度对比。

### 7.2 月份切换

所有页面最终都应支持月份切换。

当前已支持：

```text

2026年2月
2026年3月
```

后续扩展：


- 产品分析刷新
- 品类分析刷新
- 广告分析刷新
- 报告中心刷新

月度对比需要选择两个不同月份。

### 7.3 数据校验

每个月数据接入后必须校验：

- 产品 ID 是否全为 7 位数字

- 百分比是否为 0–1 或 0–100 的统一口径

- 毛利润允许为负

- 店铺名称是否在店铺字典内
- 产品名称是否为空
- 品类名称是否为空

需要输出异常清单：

- 缺失产品 ID

- 同一产品多个 ID

- 品类无法匹配

- 广告截图缺少周期

异常不应静默吞掉。

## 8. 自动结论规则

### 8.1 经营总览结论

规则示例：

- 毛利率低于平台：提示毛利率偏弱。
- 退损率高于平台：提示退损风险。
- 采购成本占比升高：提示供应链成本需复盘。
- 物流占比升高：提示物流成本异常。
- 广告占比升高但销售额下降：提示广告效率下降。

### 8.2 产品分析结论

规则示例：

- 毛利率 < 0：亏损产品。
- 退损率 > 30%：高退损风险。
- 销售额高但毛利低：规模产品但利润质量弱。
- 多店铺出单：标记多店铺产品。
- 本月新增但亏损：新增亏损增长。

### 8.3 品类分析结论

规则示例：

- 个人品类订单占比高于平台，但毛利率低：优势品类利润不足。
- 平台订单高，个人订单低：潜在扩展品类。
- 个人退损率高于平台：品类退损风险。
- 个人毛利率低于平台：品类利润弱。

### 8.4 广告分析结论

规则示例：

- ROAS 最高周期：可复盘投放策略。
- ROAS 最低周期：需要检查关键词、主图、价格。
- 曝光高 CTR 低：主图或标题弱。
- 花费 > 0 且转化 = 0：广告无效消耗。

### 8.5 月度对比结论

规则示例：

- 收入下降：提示下降比例。
- 毛利润下降幅度大于销售额下降幅度：利润质量变差。
- 退损率上升：退损需复盘。
- 新增增长产品存在亏损：单独标红。
- 某品类逆势增长：提示重点关注。
