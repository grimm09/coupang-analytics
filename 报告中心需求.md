# Coupang.JY 报告中心前端需求说明

## 1. 页面定位

报告中心用于把当前网站里的核心分析模块统一沉淀成“可查看、可筛选、可导出、可复盘”的月度经营报告入口。

当前网站已有模块：

- 经营总览
- 产品分析
- 品类分析
- 广告分析
- 月度对比

报告中心不是单独新增一套数据逻辑，而是基于以上模块的数据层生成报告。用户进入报告中心后，可以选择月份、店铺、报告范围和导出格式，快速生成月度经营报告、产品报告、品类报告、广告报告和月度对比报告。

## 2. 页面目标

报告中心需要解决 4 件事：

1. 汇总当前各分析页面的核心结论。
2. 支持按月份、店铺、口径生成报告。
3. 支持导出适合展示的报告文件。
4. 为后续 Matplotlib 月度展示报告预留接口。

## 3. 页面入口

左侧导航保留当前结构：

- coupang.JY
- 经营总览
- 产品分析
- 品类分析
- 广告分析
- 月度对比
- 报告中心

点击「报告中心」后，主标题显示：

```text
报告中心
```

副标题显示：

```text
月度经营复盘、模块报告导出与展示资料生成
```

## 4. 页面整体布局

页面保持当前网站视觉风格：

- 紫色渐变主色
- 白色圆角卡片
- 横向数据展示
- 字体大小与产品分析、经营总览保持一致
- 避免小字、密集竖排
- 适配浏览器屏幕宽度

建议页面结构：

```text
顶部筛选区
↓
报告概览 KPI
↓
报告类型卡片
↓
报告预览区
↓
导出设置区
↓
历史报告列表
```

## 5. 顶部筛选区

### 5.1 筛选项

顶部筛选区横向排列：

| 筛选项 | 默认值 | 说明 |
|---|---:|---|
| 报告月份 | 2026年3月 | 当前已支持 2026 年 1 月、2 月、3 月，后续接入 4–5 月 |
| 店铺 | 全部店铺 | 支持本土88、本土70、后续新增店铺 |
| 数据口径 | 个人产品 | 支持个人产品、平台整体 |
| 报告类型 | 月度经营报告 | 可切换不同报告模板 |

### 5.2 店铺扩展要求

当前只有两个店铺，但需要预留后续新店铺：

- 本土88
- 本土70
- 后续新增店铺

店铺筛选不要写死为两个按钮，建议数据结构使用数组：

```ts
stores = ["全部店铺", "本土88", "本土70"]
```

后续新增店铺时，只需要扩展数据源，不需要改 UI 结构。

## 6. 报告概览 KPI

顶部 KPI 横向展示，作为报告中心的快速状态判断。

建议展示 6 个指标：

| 指标 | 数据来源 | 展示要求 |
|---|---|---|
| 可生成报告数 | 当前已接入模块数量 | 例如 5 份 |
| 当前报告月份 | 月份筛选 | 例如 2026年3月 |
| 销售额 | 经营总览 / 产品分析 | 金额必须带 ¥ |
| 订单量 | 经营总览 / 产品分析 | 展示整数 |
| 毛利率 | 经营总览 / 产品分析 | 百分比 |
| 风险项数量 | 产品分析、广告分析、月度对比 | 高退损、低毛利、广告异常等 |

### 6.1 金额展示规范

所有金额必须使用人民币符号：

```text
¥18.4万
¥183,570
¥27,807
```

不要写：

```text
人民币销售额
人民币毛利润
```

字段名称只写：

- 销售额
- 毛利润
- 客单价
- 变化额

## 7. 报告类型卡片

报告中心需要提供 5 个报告类型卡片，横向或两行卡片展示。

### 7.1 月度经营报告

来源模块：经营总览。

需要包含：

- 订单量
- 销售额
- 商品数 / 商品销量
- 客单价
- 退损率
- 毛利率
- 采购成本占比
- 物流占比
- 广告占比
- 个人品类和平台品类对比

适合导出为：

- PDF
- PNG 长图
- Matplotlib 展示图

### 7.2 产品分析报告

来源模块：产品分析。

需要包含：

- 订单量
- 销售额
- 商品数
- 商品排行（销售额）
- 产品 ID
- 产品名称
- 店铺标记：本土88 / 本土70 / 本土88/70
- 多店铺出单标记
- 销售额排行
- 毛利率排行
- 退损异常产品
- 低毛利产品

表格要求：

- 「销售额、订单、客单价、毛利率、退损」这些排行数据居中。
- 产品名称过长可以省略号。
- 产品 ID 必须是正常 7 位数。

### 7.3 品类分析报告

来源模块：品类分析。

需要包含：

- 个人品类总览
- 平台品类总览
- 个人品类出单订单量
- 平台品类出单量
- 个人 & 平台出单品类对比
- 品类销售额
- 品类毛利率
- 品类退损率
- 品类成本结构

图表要求：

- 饼图文字不能与图形重叠。
- 饼图中心文字必须居中。
- 品类标签拥挤时优先使用图例或右侧列表，不强行全部贴在饼图外圈。

### 7.4 广告分析报告

来源模块：广告分析。

需要包含：

- 曝光量
- 点击量
- CTR
- CPC
- 广告订单数
- 广告转化销售量
- CVR
- 广告花费
- 广告销售额
- 总销售额
- ROAS
- 广告销售占比
- 分周期广告表现
- 广告异常提示

广告分析报告需要保留后台截图数据来源说明：

```text
数据来自广告后台截图
```

### 7.5 月度对比报告

来源模块：月度对比。

当前默认跟随网站默认月份，已为：

```text
2026年2月 vs 2026年3月
```

当前已支持：

```text
2025年12月 vs 2026年1月
2026年1月 vs 2026年2月
2026年2月 vs 2026年3月
```

需要包含：

- 收入合计
- 有效订单量
- 商品销量
- 毛利润
- 毛利率
- 退损率
- 个人产品双月柱状对比
- 个人收入环比瀑布图
- 个人成本结构对比
- 品类月度对比
- 产品变化榜：收入增长 TOP10
- 产品变化榜：收入下滑 TOP10
- 默认结论

月度对比报告必须使用个人产品口径作为默认展示。

## 8. 报告预览区

报告预览区是报告中心的核心区域。

用户选择报告类型后，下方展示对应报告的预览卡片。

### 8.1 默认展示

默认展示「月度经营报告」预览。

预览内容包括：

1. 报告标题
2. 报告月份
3. 数据口径
4. 核心 KPI
5. 关键图表缩略预览
6. 自动结论
7. 导出按钮

### 8.2 预览卡片结构

建议结构：

```text
报告预览
├─ 标题：2026年3月经营复盘报告
├─ 标签：个人产品 / 全部店铺 / 2026年3月
├─ KPI：订单量、销售额、毛利率、退损率
├─ 图表缩略：趋势图、品类图、产品榜
├─ 自动结论：3–5 条
└─ 操作：查看详情、导出报告
```

## 9. 自动结论规则

报告中心需要根据已有模块生成默认结论。

### 9.1 经营总览结论

示例规则：

- 退损率高于平台整体，提示「退损率偏高，需要复盘退损来源」。
- 毛利率低于平台整体，提示「毛利率低于平台，需要关注成本结构」。
- 广告占比升高但销售额未同步增长，提示「广告效率需要复盘」。

### 9.2 产品分析结论

示例规则：

- 产品毛利率小于 0，标记为「亏损产品」。
- 退损率高于 30%，标记为「高退损风险」。
- 同一产品在多个店铺出单，标记为「多店铺出单」。
- 销售额高但毛利低，标记为「规模产品但利润质量偏弱」。

### 9.3 品类分析结论

示例规则：

- 个人品类订单占比高于平台，但毛利率低，标记为「优势品类但利润不足」。
- 平台出单高而个人出单低，标记为「潜在扩展品类」。
- 个人退损率高于平台，标记为「品类退损风险」。

### 9.4 广告分析结论

示例规则：

- ROAS 低于目标值，标记为「广告效率偏低」。
- 曝光高但 CTR 低，标记为「主图 / 标题点击弱」。
- 有花费但无转化，标记为「广告无效消耗」。
- 后半月 ROAS 上升，标记为「可复盘投放策略」。

### 9.5 月度对比结论

示例规则：

- 收入下降，提示下降幅度。
- 毛利润下降幅度大于销售额下降幅度，提示利润质量变差。
- 退损率上升，提示退损复盘。
- 新增增长产品中若存在亏损，提示单独标红。

## 10. 导出设置区

报告中心需要提供统一导出设置。

### 10.1 导出格式

第一阶段支持：

- PDF
- PNG 长图
- CSV / Excel 数据表

第二阶段预留：

- Matplotlib 月度展示图
- PPT / 演示稿

### 10.2 导出内容选择

用户可以勾选导出内容：

- 封面
- 核心 KPI
- 经营总览
- 产品分析
- 品类分析
- 广告分析
- 月度对比
- 自动结论
- 明细表格

默认勾选：

- 封面
- 核心 KPI
- 经营总览
- 产品分析
- 品类分析
- 广告分析
- 月度对比
- 自动结论

明细表格默认不勾选，避免报告太长。

### 10.3 导出按钮

按钮文案：

```text
生成报告
导出 PDF
导出 PNG
导出数据表
```

主要按钮使用紫色渐变。

## 11. 历史报告列表

报告中心底部展示历史报告。

### 11.1 字段

| 字段 | 说明 |
|---|---|
| 报告名称 | 例如 2026年3月经营复盘报告 |
| 报告月份 | 例如 2026年3月 |
| 报告类型 | 月度经营 / 产品分析 / 广告分析等 |
| 数据口径 | 个人产品 / 平台整体 |
| 店铺 | 全部店铺 / 本土88 / 本土70 |
| 生成时间 | 报告生成时间 |
| 状态 | 已生成 / 生成中 / 失败 |
| 操作 | 预览 / 下载 / 重新生成 |

### 11.2 状态设计

```text
已生成：绿色
生成中：紫色或蓝色
失败：红色
```

## 12. 数据来源

报告中心复用当前项目数据层。当前已接入 2026 年 1 月、2 月、3 月，不应只读取 january* 数据。

| 模块 | 数据来源 |
|---|---|
| 经营总览 | januaryOverview / februaryOverview / marchOverview |
| 产品分析 | januaryProductAnalysis / februaryProductAnalysis / marchProductAnalysis |
| 品类分析 | januaryCategoryAnalysis / februaryCategoryAnalysis / marchCategoryAnalysis |
| 广告分析 | januaryAdAnalysis / februaryAdAnalysis / marchAdAnalysis |
| 月度对比 | monthCompare / monthCompareFebruary / monthCompareMarch |

后续接入 4–5 月数据并整理数据层后，建议统一改为：

```ts
monthlyData = {
  "2026-01": {...},
  "2026-02": {...},
  "2026-03": {...},
  "2026-04": {...},
  "2026-05": {...},
}
```

报告中心只根据用户选择的月份读取对应数据。

## 13. Matplotlib 月度报告预留

报告中心需要预留 Matplotlib 导出接口。

### 13.1 使用场景

Matplotlib 用于生成更适合月度展示的静态图：

- 月度经营汇总图
- 成本结构图
- 品类对比图
- 产品增长 / 下滑榜
- 广告周期表现图

### 13.2 导出位置

建议第一阶段导出到本地项目目录：

```text
outputs/coupang-analytics/reports/
```

后续可以在个人网站的报告中心中提供下载入口。

### 13.3 前端展示方式

前端报告中心只负责：

- 触发生成
- 显示生成状态
- 展示图片预览
- 提供下载按钮

Matplotlib 实际生成逻辑可以由后端脚本或本地 Python 脚本完成。

## 14. 视觉要求

报告中心必须延续当前页面视觉：

- 左侧导航固定。
- 主内容使用大圆角浅紫背景容器。
- 卡片使用白色背景、轻阴影。
- 图表和数据横向排列。
- 字体不要过小，保持与产品分析调整后的字号一致。
- 表格文字居中，尤其是金额、订单、毛利率、退损率。
- 长名称使用省略号。
- 关键风险用红色标签。
- 正向增长用绿色标签。
- 默认按钮使用紫色。

## 15. 首版开发范围

首版报告中心先做视觉和交互框架，不需要一次性完成所有真实导出。

### 必须完成

- 报告中心页面入口
- 顶部筛选区
- 报告概览 KPI
- 5 个报告类型卡片
- 月度经营报告预览
- 导出设置卡片
- 历史报告列表静态框架

### 可以暂缓

- 真正生成 PDF
- 真正生成 PNG 长图
- 真正生成 Excel
- 真正调用 Matplotlib
- 报告历史持久化

## 16. 首版页面默认内容

默认进入报告中心时：

```text
报告月份：2026年3月
店铺：全部店铺
数据口径：个人产品
报告类型：月度经营报告
```

默认展示报告标题：

```text
2026年3月 Coupang.JY 经营复盘报告
```

默认展示 4 条自动结论：

1. 3月销售额为 ¥22.8万，订单量为 2,879。
2. 毛利率为 19.81%，较 2 月下降 4.21pp，需要关注成本结构。
3. 退损率为 6.01%，较 2 月继续改善。
4. 产品、品类、广告与月度对比模块已接入；3 月广告来自本土88+本土70广告后台截图识别，可生成完整月报。

## 17. 验收标准

首版验收时需要确认：

- 报告中心可以从左侧导航打开。
- 页面风格和当前经营总览、产品分析一致。
- 数据展示横排，不做密集竖排。
- 金额全部带 ¥。
- 页面字体不小于现有产品分析页面。
- 报告类型卡片能清楚看出 5 个模块。
- 导出区能看出后续 PDF、PNG、Excel、Matplotlib 的方向。
- 历史报告列表结构完整。
- 浏览器宽度变化时不会出现严重重叠。
